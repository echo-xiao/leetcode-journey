# 83. 删除排序链表中的重复元素

**难度**: Easy | **标签**: `Linked List`

## 题目描述

<p>给定一个已排序的链表的头<meta charset="UTF-8" />&nbsp;<code>head</code>&nbsp;，&nbsp;<em>删除所有重复的元素，使每个元素只出现一次</em>&nbsp;。返回 <em>已排序的链表</em>&nbsp;。</p>

## 解题思路与复盘

1. **一句话直击本质：** 通过遍历链表，比较当前节点与下一个节点的值，若相同则跳过下一个节点以删除重复元素。

2. **综合思路：**
   - **迭代法：** 所有版本均采用迭代法，通过一个指针遍历链表，逐一比较相邻节点的值，若相同则调整指针以跳过重复节点。
   - **递归法：** 虽然在提供的代码集中没有递归实现，但递归法也是一种常见的思路，通过递归调用处理子链表，合并结果时跳过重复节点。

3. **全量伪代码：**
   - **迭代法伪代码：**
     ```
     如果链表为空或只有一个节点，返回头节点
     初始化当前节点为头节点
     当当前节点和下一个节点都不为空时：
         如果当前节点的值等于下一个节点的值：
             跳过下一个节点（即当前节点的 next 指向下下个节点）
         否则：
             移动当前节点到下一个节点
     返回头节点
     ```
   - **递归法伪代码（假设）：**
     ```
     如果链表为空或只有一个节点，返回头节点
     递归处理子链表（从下一个节点开始）
     如果当前节点的值等于递归返回的头节点的值：
         跳过重复节点（即当前节点的 next 指向递归返回的头节点的 next）
     返回当前节点
     ```

4. **复杂度：**
   - **时间复杂度：** $O(n)$，其中 $n$ 是链表的节点数，因为每个节点最多被访问一次。
   - **空间复杂度：** $O(1)$，因为只使用了常数个额外指针。

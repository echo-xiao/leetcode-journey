# 1149. 三个有序数组的交集

**难度**: Easy | **标签**: `Array` `Hash Table` `Binary Search` `Counting`

## 题目描述

None

---
## 解题思路与复盘

1. 一句话直击本质：使用三个指针分别遍历三个有序数组，通过比较指针指向的元素值来同步推进指针，找到三个数组的交集。

2. 综合思路：
   - **双指针/多指针法**：利用三个指针分别指向三个数组的当前元素，通过比较指针指向的元素值，决定哪个指针需要移动，以此来同步推进，直到找到相同的元素。
   - **集合交集法**（未在代码集中出现）：可以通过将每个数组转换为集合，然后使用集合的交集操作来找到公共元素。

3. 全量伪代码：
   - **多指针法**：
     ```
     初始化结果列表 res
     初始化指针 p1, p2, p3 为 0
     当 p1, p2, p3 都在各自数组范围内时：
         取出 arr1[p1], arr2[p2], arr3[p3] 的值为 val1, val2, val3
         如果 val1, val2, val3 相等：
             将 val1 添加到结果列表 res
             p1, p2, p3 都加 1
         否则：
             如果 val1 < val2 或 val1 < val3：
                 p1 加 1
             如果 val2 < val1 或 val2 < val3：
                 p2 加 1
             如果 val3 < val1 或 val3 < val2：
                 p3 加 1
     返回结果列表 res
     ```

4. 复杂度：
   - 时间复杂度：$O(n_1 + n_2 + n_3)$，其中 $n_1, n_2, n_3$ 分别是三个数组的长度，因为每个指针最多遍历各自数组一次。
   - 空间复杂度：$O(1)$，除了输出结果外，使用的额外空间是常数级别的。
